# くらしの仕組み化アプリ 🌱

**意志力を1ミリも要求しない**、行動トリガー型の習慣化アプリです。
「めんどくさい」「疲れた」が勝っても続くように、行動科学の知見をUIの形に落とし込んでいます。

- **完全に端末内だけで動きます**（アカウント不要・データは自分のスマホの中だけ）
- **軽量なJavaScript**で書かれた静的サイト。サーバーは使わず、**開いた瞬間に立ち上がります**（スマホ・モバイル通信でも快適）
- **GitHub Pages** で無料公開でき、**ホーム画面に追加**すればアプリのように使えます

> ℹ️ もともとはPython(PyScript)版でしたが、スマホでの初回読み込みが重かったため、見た目・機能はそのままに軽いJavaScriptへ作り替えています。

---

## 📱 スマホでの使い方（いちばん大事）

### ① 開く
公開したURL（後述の手順で作ります）をスマホのブラウザで開きます。
🌱の画面は一瞬だけ出て、すぐにホームが立ち上がります。

### ② ホーム画面に追加する（これで“アプリ”になります）
- **iPhone（Safari）**：画面下の「共有」ボタン（□に↑）→ **「ホーム画面に追加」**
- **Android（Chrome）**：右上の「⋮」→ **「ホーム画面に追加」**

これで全画面表示になり、オフラインでも起動できます。

### ③ 通知について（任意）
アプリ内の「設定 → 通知 → 通知を許可する」で有効にできます。
- **iPhoneは「ホーム画面に追加」した後でないと通知できません**（Safariの仕様）。
- 通知はあくまで保険です。**このアプリは通知に頼りません** — 開いた瞬間、画面のいちばん上に「いま、これだけ」が大きく出るので、それをタップするだけです。

## 🕹 使い方（3ステップ）

1. **初回**：なりたい自分を1つ選ぶ → 習慣を確認（そのままでOK）→ はじめる（テキスト入力ゼロ）
2. **毎日**：開いて、いちばん上の大きなボタン（例：🏠 帰宅して靴を脱いだら）をタップ
3. 出てきた「やること」を見て **「できた！」** をタップ。以上です

疲れた日は上部の **「🔋 最小モード」** をONにすると、全部が2分以内の最小版に切り替わります。
**最小でも達成は満額**カウントされます。

## 🚀 GitHub Pagesで公開する手順（初心者向け）

> 「GitHub（ギットハブ）」は、ファイルを置いて無料でWEB公開できるサービスです。以下は一度きりの作業です。

### 1. GitHubアカウントを作る
https://github.com/signup で無料登録します（メールアドレスとパスワードだけ）。

### 2. 新しいリポジトリ（＝置き場所）を作る
- 右上の「＋」→ **New repository**
- Repository name に例えば `kurashi-app` と入力
- **Public**（公開）を選択
- 「Create repository」を押す

### 3. ファイルをアップロードする
作成した画面の **「uploading an existing file」** リンクをクリックし、
この `kurashi-app` フォルダの**中身すべて**（`index.html` など）をドラッグ＆ドロップ →
下の緑ボタン **Commit changes** を押します。

> ※ `serve.py` は自分のパソコンで動作確認する時だけ使う開発用ファイルです。アップロードしてもしなくてもアプリの動作には影響しません。

### 4. Pagesを有効にする
- リポジトリ上部の **Settings** → 左メニューの **Pages**
- **Source** を **Deploy from a branch** にし、Branch を **main / (root)** にして **Save**
- 1〜2分待つと、上部に公開URL（`https://ユーザー名.github.io/kurashi-app/`）が表示されます

このURLをスマホで開いて、上の「ホーム画面に追加」をすれば完成です🎉

## 🧠 行動科学の12原則 → 実装対応表

このアプリは飾りではなく、各原則を**具体的なUI・機能**に落とし込んでいます。
（コードの詳細は `app.js` の冒頭コメントにも同じ表があります）

| # | 原則 | どこに実装したか |
|---|------|------------------|
| 1 | 摩擦の最小化 | 起動直後ホーム最上部に「いま、これだけ」を大きく表示。大トリガーボタン。操作はタップ中心 |
| 2 | 実行意図（if-then） | 習慣は必ず「〜したら→〜する」。追加も穴埋めフォーム（`buildAddHabit`）で自由記述に悩ませない |
| 3 | 習慣スタッキング | 「歯を磨いたら」「靴を脱いだら」等を`TRIGGER_PRESETS`として候補提供。自分でも追加可。記録はヒートマップで既存行動に接続 |
| 4 | 2分ルール & 最小モード | 全習慣に`actionMin`。ホームの「最小モード」で一括最小化。フロー中も1タップで最小に切替。**最小でも満額**（`completeHabit`） |
| 5 | 現在バイアス（誘惑バンドル） | 達成の瞬間に紙吹雪＋褒め言葉の演出（`fxCelebrate`）。習慣ごとに任意の「ご褒美ルール」を紐づけ可能 |
| 6 | 変動報酬 | `completeHabit`内で低確率にボーナス点・レア褒めメッセージ・レア称号が出る |
| 7 | 損失回避 & 保有効果 | ストリークを「育つ植物」で可視化（`plantFor`）。**枯れる・死ぬ演出は禁止** |
| 8 | 進捗の付与効果 | 初期ポイント20＝レベルゲージ20%から開始。スタンプカードも2個押した状態でスタート |
| 9 | フレッシュスタート & セルフコンパッション | 途切れても責めない文言（`freshStartBanner`）。**「サボり」という言葉は一切不使用**。月1回の「おやすみ券」でストリーク保護。週初・月初に再開を提案 |
| 10 | デフォルト効果 & 選択肢削減 | プリセット習慣を有効化済みで同梱（`PRESET_HABITS`）。設定を開かなくても完結 |
| 11 | アイデンティティベース | オンボで「なりたい自分」を選択。達成のたびに「今日のあなたは『◯◯な人』の行動を1つ積みました」と言語化 |
| 12 | ツァイガルニク効果 | ホームに「あと〇つで今日コンプリート」を**圧を与えない控えめなトーン**で表示 |

## 📁 ファイル構成

| ファイル | 役割 |
|----------|------|
| `index.html` | 入口。ローディング画面＋app.js読み込み＋Service Worker登録 |
| `app.js` | アプリ本体（JavaScriptの全ロジック。冒頭に原則対応表コメント） |
| `style.css` | 見た目（スマホ縦・片手・ワンタップ最優先。ダークモード対応） |
| `manifest.webmanifest` | PWA設定（ホーム画面追加で全画面化） |
| `sw.js` | Service Worker（オフライン起動＋通知の土台） |
| `icon.svg` / `icon-192.png` / `icon-512.png` / `apple-touch-icon.png` | アイコン各種 |
| `serve.py` | ローカル動作確認用の簡易サーバー（開発時のみ・公開には不要） |

## 💾 データについて
- 保存先はこの端末の中（`localStorage`）だけ。サーバーには何も送りません。
- 機種変更やバックアップ用に、設定画面から **JSONの書き出し / 読み込み** ができます。

## 🛠 自分のパソコンで動作確認する（任意）
Macのターミナルでこのフォルダに入り、次を実行して `http://127.0.0.1:4173/` を開きます。

```bash
python3 serve.py
```

（`file://` で直接開くとService Workerが動きません。必ずサーバー経由で開いてください。）

## 🌿 続けるコツ（アプリ外の話・でも一番効きます）
- アプリの前に**環境の摩擦を減らす**とさらに効きます（参考書を机に開きっぱなし、掃除道具を出しっぱなし等）。
- 最初の2週間は「簡単すぎて物足りない」くらいが正解。物足りなさは続いている証拠です。
- 習慣を増やすのは**3週間目以降**がおすすめ。
